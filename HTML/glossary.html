Glossary
Record   Translation   Database   Automation   Options

TIP: Use CTRL+F to quickly find the text you are searching for.

RECORD

Record: Every element in the game is considered as a record. From a sword to an actor or a climate.
Name: The name of the record shown ingame to the player. Example: Solitude Sawmill.
FormID: Is an unique eight digit hexadecimal code that identifies a record. Various records can share the same name but not the same formID. Example: 000161E9 There are two types:
  • BaseID: The FormID of a unique base object in the editor.
  • RefID: The FormID of every object in the game. A BaseID object can have several RefID objects.
ID: The FormIDs managed by this program are always RefIDs, never BaseIDs.
EDID: A subrecord used as a more descriptive name of a record, to be easily identified by the developers and in some modding tools. Sometimes they are a number, others mnemonics or a full name. It is used internally by the game engine, and hidden to the player ingame. Example: SolitudeSawmill. NOTE: In Morrowind, the DIAL and CELL grups are the text to translate. This is why it was so hard to translate a mod back in that time.
GRUP: A group of records. Can contains subgroups. It makes the records organized and improves the scanning speed of files.
Field (Not confuse with a DB Field): A "slot" for a subrecord. Take a look here.
Index: Each text in a mod belongs to a record, which belongs to a GRUP. But, for some of this groups, there can be various texts for a same record: same GRUP, ID, EDID and Field. So, in this case, the only way to differentiate them is by the order in which they appears. This order is used by the application to make this Index.
CRC (Cyclic Redundancy Check): An error detecting code that identifies every plugin/mod in Bethesda games, and used to check the integrity of the file and detect conflicts between them.

TRANSLATION

Main Window: The window shown when the application starts, where the translation tasks are held and from which the user access all the tools.
Table Grid: The table frame that provides access to all translatable text lines of the mod.
Panel: Refers to certain frames inside an application window showing information or offering interaction.
Menu: Refers to any of the dropdown menus in the Menu bar of the Main Window.
Toolbar: Refers to the icon or information bars of the Main Window.
Field (Not confuse with a record Field): Any window or table frame containing data, editable or not, in the program.
Cell: Each one of the data squares of a table grid.
Line: Each one of the horizontal group of cells of a table grid.
Selected Text: This occurs when the text inside a cell is selected by clicking two times on the cell.
Selected Line/Cell/Field: This occurs when a Line/Cell/Field is selected by clicking one time on them, or just selecting them with the arrow keys.
Comments: This panel appears in various windows of the program. It is fully editable and has different functions depending on what type of cell is selected. These are some examples:
  • Normal Text Line: Useful for team or personal notes, to remember or clarify something about the line.
  • Script Code Line: In Skyrim and Fallout 4, it will show the lines of the script related to the text in the selected field, to help the user to identify visually what and what not to translate. For older games, it will show only the function in which appears the line. WARNING: Do not translate directly here. Modify the code wrongly in this field can corrupt the script and produce important problems if used in a game.
  • NPC Name Line: It will show the race and sex of the NPC.
  • INFO lines: Shows additional data in the dialog lines such as the related quest, the emotion with which it is expressed, the linked Audio filename, etc.
Context menu: The menu shown after right click on the table grid in the Main Window.
Analyze: A task performed by the application to detect the structure of a mod or script, and extract the data needed to translate.
Compare: A task performed by the application to detect similarities between the data extracted from the mod and the databases.
The comparison order is the following:
1. Main DB
2. Custom DB
3. OnlyText DB
4. Dictionary without numbers
5. Dictionary without punctuation
Translate: A task performed automatically by the application or manually by the user, converting the text of a cell or cells from one language to another. When it is done automatically, it occurs after a comparison with one or more databases.
Validate: Saves the changes made in the database persistently, to be valid when required.
Loaded Mod: The mod actually opened to be translated in the application. It is loaded by the Open File (Ctrl+O).

DATABASE

DB: Acronym for database.
Instant translation: Allows to apply databases individually or collectively to the current translation.
Mass translation: A simple tool to extract quickly the data of several mods. Just drag and drop the mods in the window and hit on "Start Translation".
Search Database: Allows to consult and edit five types of databases.
Main Database: A database containing all the original texts extracted from a game.
Custom Database: A database similar to the Main Databases, offered as an alternative for the user to create his own.
TextOnly Database: A ligth database tool that has less features, but allows to contain more data and manage it faster.
Mod Database: A temporal database created from the current loaded mod.
External Database: A temportal database (cannot be saved) that allows to load ANY type of database and create new ones from many types of game files.
Personal EET/XML Database: The database created from the current mod when the user saves the translation. It is created in the folder Bases Persos inside the installation folder of the game. It can be used to continue the translation later and to translate new versions of the mod.
Load an already translated mod: Allows to select a translated mod for the application makes relations to translate the matching lines, based on GRUP+ID+EDID+Field+Index. In this case, no verifications are done on the Original Text, because there is not a main database for that text, and the application can not know the changes. This forces the user to revise the translation to correct manually possible mistakes. This is a common situation when translating a newer version of a mod, from an older translated version.
Importation: A tool used mainly to create/modify modularly databases importing text data from games/mods and to swap the order of the languages in a bilingual database.
Fast Import: A tool used mainly to quickly create/modify modularly big databases importing text data from games/mods and to swap the order of the languages in a bilingual database.
Hunspell: (from Wikipedia) A spell checker and morphological analyzer designed for languages with rich morphology and complex word compounding and character encoding,
BDD: Is the acronym for "Base de données" in French, that means "Database" in English. In the installation folder of the program the is as BDD folder containing the Main, Custom, TextOnly and External databases downloaded or created by the user.
%VAR%: A code tag to help the program to detect a variable text/word in a specific place of several sentences, that exist in the databases available, and translate them automatically. It is used in several of the Automatic Translations tools of the program.
%PARAM%: A code tag to allow the user to indicate a list of parameters that change in several lines, to help the program to translate them automatically. It is only used in Automatic Translation 1.
Dictionary without punctuation: Each database have an internal dictionary of text without punctuation, used by several tools of this program to autotranslate the text of the mods skipping punctuation variations. For example:

This dialog sentence...
        "Hello! How are you?"
is stored in the dictionary as...
        "hellohowareyou"
so, when this another sentence is translated...
        "Hello! How are you!!!"
uses the same dictionary entry stored, because both phrases share the same text without punctuation.

Dictionary without numbers: Each database have an internal dictionary of text without numbers, used by several tools of this program to autotranslate the text of the mods skipping number variations. For example:

This dialog sentence...
    "Give me 6 or 8 eggs!" is stored in the dictionary as...
    "Give me %var01% or %var02% eggs" so, when this similar sentence appears...
    "Give me 45 or 80 eggs" it is translated correctly.

Instead, if you translate the...
    "Give me 6 eggs!" any sentence matching this pattern...
    "Give me XX eggs!"
will be translated directly at the same time, without being added to any dictionary. In this case, the auto translation, happens just translating a line.

AUTOMATION

Automatic Translation (also Auto Translation): A tool implemented to solve quickly and easily some special translations patterns that can be found in some mods. The program includes some of these tools.
AT: Acronym for Automatic Translation.
Clone Auto Translation: A hidden tool that translates automatically all the lines (in the Main Window), containing exactly the same text as other translated manually.
Numerical & Punctuation Autotranslation: A hidden tool that Translates automatically the lines matching exactly the same text without accounting any numerical or punctuation variations, after one of the lines is translated manually. It uses the same technique than the without numbers and punctuation dictionaries, but it this case it only uses the text typing text, not the databases.
Carriage Return: A control (hidden) character used to finish/start a line in a text field or document. It is inserted by pressing the key Enter, but, as said, it is not visible for the user.
Pattern: A text scheme common in two or more sentences, that can't be translated automatically without the use of the Automatic Translation tools offered by the program.
Rule Pattern: Any scheme of text/words used in any of the Automatic Translation tools.
Ignored Texts: A tool used to indicate the program any text to be ignored while translating. For example, the books in Skyrim include some layout code that should not be translated or modified. So, indicating the code/s in this tool, the program will ignore that text, translating the rest of the text.

OPTIONS

Character Encoding: A group of characters (mainly grapheme [abc...] and symbols ['&#...]) preset to be used in a determined context, for example, the common letters and symbols used to represent a language or a group of similar languages, like those that come from Latin. The ones included in this program are listed below:
UTF-8: Unicode Transformation Format – 8-bit. An universal character encoding containing all possible characters.
Windows-250: A character encoding for Central European and Eastern European languages that use Latin script (Polish, Hungarian...).
Windows-251: A character encoding for Cyrillic script (Russian, Bulgarian...).
Windows-252: A character encoding of the Latin alphabet (English and some other Western languages).
Shift_jis: A character encoding for Japanese language. It is used for old games, like Oblivion.
Zlib compressor level: In a mod, some entries are compressed by Zlib.
