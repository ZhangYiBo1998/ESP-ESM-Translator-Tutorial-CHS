FILE MENU

Open (esp/esm) file
Opens a dialog box to select the mod to translate. It also shows the recently opened files to launch them quickly. It also offers to open a previously opened mod to translate by choosing one from the list, when the mouse is on it.

Open Archives (bsa, ba2, erf...)
NOTE: The BA2 format support is full for SSE, but for FO4 the extraction of images is not implemented.
 Opens a dialog box to select an archive to open in the browser. It also includes an internal viewer that allows to preview texture images (.dds) except the Fallout 4 ones, source scripts (.psc), sound files (.fuz and others), and text files (.txt). By double clicking on these and other not supported files in the preview, like the Nifscope (.nif) ones, they will be opened directly with the system default program associated to the file extension. It also shows the recently opened files to launch them quickly.

_WINDOW_
Research: Allows to quickly find files (not folders) by introducing text in the field. Then by pressing Enter, the next file containing the text will be selected, and so on.
Tree View: Allows file/folder visualization, navigation and selection in a tree way.
Extract Selection: Allows to extract the selected files of the compressed archive after press the button.
Extract all: Allows to extract all the files of the compressed archive.
Close: Closes the archive window.

_HOWTO_
Press on Open Archives (bsa, ba2, erf...), select a compatible archive (bsa, ba2, erf, dazip, lvl) in your system, and press Open. Now you can just visualize internally one file by selecting it in the Tree View, open it in an external edition tool by double click, or extract the files globally by pressing Extract All, or extract just a selection of them, or just one by selecting them by the mouse while the CTRL key is pressed and pressing on Extract Selection button. After finish your tasks press the Close button.

Launch the translation of the mod
Starts the translation of the mod applying the validated lines into the ESP/ESM. A dialog box will open to select the location to save the translated mod. A default destination folder can be set in the Options window for Bethesda games.

Options
Opens the Options window to setup the application.

Language
Allows to translate this application to another language by Add language and edit the existing ones in the table grid. Each language file contain all the text used by the program, included the interface, options and tooltip texts. The button Delete language deletes an existing language. The Filter bar at top of the windows, is useful to search and focus on a specific text cell. After introduce changes to the database. it is needed to validate them by pressing Validate Modifications.
The Import a file option is there for the author of the program or for a former translator, to import new translations from other users into the Database language. If you translate the program into a new language, do not hesitate to send to Epervier 666 the lang.xml file, to integrate it in the program and make it available to others.

Close current mod
Closes the active mod translation tab.

Exit
Closes the program.

TRANSLATION MENU

Save Translation (Ctrl+S)
Allows to save the current opened mod translation table as a personal EET/XML database, which can be reloaded to be modified later or used as a translation file for other versions or mods. The first time it is saved, will offer the user the possibility to change the default name of the save, and the location to save it. Then, it will be saved automatically. To change them onwards, the Save Translation As... option (below) should be used.
NOTE: Older versions of the program used three different XML files, to save a translation: one for the mod, other for the scripts and another for the MCM menus. This no longer occurs, and now is used a single XML file for all.

Save Translation As...
Allows to save the translation offering always the possibility to change the default name of the save, and the location to save it.

Save Translation Partially
Allows to select the parts of the translation to be saved, filtering them by GRUP and by the way the lines where translated. Also, the Save lines selected in the table checkbox allows to save only the lines selected in the Table Grid of the Main Window, if checked.
This is useful for teamwork, allowing to each member of the team to save the part/s they are working on, and then merge them when all parts are finished, without overlaps or conflicts.

Load Translation (Ctrl+L)
Allows to load a personal EET/XML database previously saved by the Save Translation option above, or downloaded from other users, to be modified or used as a translation file for other versions or mods. It´s perfectly viable to load a file which does not correspond with the mod currently loaded, because only the matching elements will be used. You can share a saved file with other users, so they will be able to use it to translate its mod.

Do comparison with Database
Compares the current mod loaded with the Main Database, and translates automatically the lines not modified manually that corresponds with it. This is made automatically every time a mod is loaded, but sometimes can be needed to relaunch it, after edit manually the Main Database, after change the Main Database in the Databases tab of the Options window or after delete the text translated in one or more cells to restored it again. They are not common scenarios though, but may occur.

Do comparison with Custom Base
Same as previous, but using the Custom Database.

Do comparison with TextOnly Base
Same as previous, but using the TextOnly Database.

Load an already translated mod
Allows to select a translated mod for the application makes relations to translate the matching lines, based on GRUP+ID+EDID+Field+Index. In this case, no verifications are done in the Original Text, because there is not a main database for that text, and the application can not know the changes. This forces the user to revise the translation visually to correct manually possible mistakes. This is a common situation when translating a newer version of a mod, from an older translated version.

Window for Scripts
This is a special window to translate scripts. It is also possible to translate them directly in the Main Window, but this tool offers more information and Syntax Highlighting for the Papyrus code.

Some of the panels and fields here, like the Filter Bar, the Table Panel, the Original, Translated & Comments fields, are similar to the Main Window, but others are exclusive, like the left Scripts List Outliner, which allows to select each script individually, and the Full Script Viewer at bottom, which allows to see the full script.

The Status Bar offers some additional information like the Scripts Count, the number of non empty lines, and the number of translated lines.

Autotranslation Options

IMPORTANT: It is needed to have selected the right database/s in the Databases tab of the Options window. to get any of the five autotranslation utils working properly.

Automatic Translations are a group of tools implemented to solve quickly and easily special translation text patterns that can be found in mods. Without these tools, the manual work and time invested would be huge in complex cases, but with them the results are achieved instantly.

The program offers a few of these patterns predefined, like examples, for some languages. But the more important is the possibility to create new autotranslation files and entries, because each mod has its own patters to resolve and they require the user action to create them manually to apply them later automatically.

When they are triggered by the Apply Automatic Translations (F9) or individually in the Edition Menu, the program compares the autotranslation patterns with all databases and the current mod, resolving the matching ones and translating automatically all the lines affected. The order in which it searches the databases is indicated in the Miscellaneous Tab of this Autotranslation window, and can be modified by drag and drop them.

Also, bear in mind that the AT2 and AT3 are disabled by default, because now can be replaced by AT4 and AT5 respectively. Including AT1 can be replaced by AT5, and the future of these three tools is not clear inside this application. Anyway, they are operative and can be used if the user prefer to use them. In fact, they have some configuration details that are not present in AT4 and AT5, at least at this moment, but they are not important though.

Auto Translations 1 (AT1)

NOTE: Now the AT1 tasks can be done with AT5 without the limitation of GRUP/FIELD. The %PARAM% in AT1 can be replaced by %VARXX% in AT5 (XX must be changed by the number of the variable).

This tool allows to translate automatically several special multiline patterns that contains two or three of these elements:

TEXT: A part of invariable text repeated in various lines that is found (or not) in the databases available. It is defined manually by the user when creates the rule pattern.
%VAR%: A part of variable (or not) text in the same place of several lines that exist in the databases available. It is defined as %VAR% in the rule pattern.
%PARAM%: A part of changing text parameters in the same place of several lines, that exist in the databases available. It is defined as %PARAM% in the rule pattern.

They can be found in two or more element patters, but can only contain one %VAR% and one %PARAM% by pattern. These are some of the possible combinations:

Text %VAR%
%VAR% Text
 Text %VAR% Text
%VAR% %PARAM%
%VAR% Text %PARAM%
...

TIP: To solve patterns that need more than one variable, use AT5.
_HOWTO_
In the AT1 Options window, there are just three configurable elements:
1. Automatic Translation File: It is a dropdown selection list, allowing to choose between a group preconfigured special patterns for different languages, and to create a new group for another language or another group for an existing language. To do the later, just press on <new>, choose a name for the new special pattern group and press OK.
2. Automatically Apply: If checked, the changes will be applied automatically when a mod is translated. For example, while translating "Hello Argonian" by "Bonjour Argonien", the application will detect that Argonian is a race name included in AT4, so it will start to search other similar lines with race entries (like Aldmer, Breton, etc.), translating them automatically. It won´t try to search "Hello Daedric" because Daedric is in a group "Metal" and Argonian is in the group "Race".
3. A table grid with two columns:
Active: Allows to check/uncheck the special pattern selected, to activate/disable it.
Comments: This is a descriptive "tittle" for every configured special pattern.

To create a new line, just right click on the lines frame and choose Add line. The configuration window will open to fill all the fields needed for the pattern. To modify an existing line, right click on the desired line and choose Modify line or just double left click on it, and the same configuration window will open to tweak it. In the right click context menu on the lines frame, there are two options to Delete line and Copy line, to perform these basic tasks.

_WINDOW_
Tittle: The top empty field is to introduce the comment "tittle" of the pattern, to be easily identified in the lines frame.
Activated: Allows to check/uncheck the special pattern selected, to activate/disable it.
Trigger GRUPs/FIELDs: Allows to filter the GRUPs/FIELDs allowed to trigger the translation in the Original Text field.
Application GRUP/FIELD: Allows to filter the GRUPs/FIELDs that will be applied in the translation of the Translated Text field.
Original Text: Field to indicate the original language text pattern to translate, using any combination of the three elements: TEXT, %VAR% and %PARAM%.
Translated Text: Field to indicate the translated text pattern, using any combination of the three elements: TEXT, %VAR% and %PARAM%.
%VAR%: Tag to indicate a variable text in the pattern. The only condition is that the text must be translated in the databases.
%PARAM%: Tag to indicate parameters. Commonly they are variable adjectives or numbers affecting a repeated text part of the sentence. These parameters are not in the databases, at least individually, therefore, they must be specifies in the two parameters fields below.
Original parameter(s): Allows to indicate the parameters to translate. If there is more than one, they must be separated by comma.
Translated parameter(s): Allows to indicate the translated parameters. If there is more than one, they must be separated by comma.
Try to use parameters if original field use it: Activates/Deactivates the use of parameters if they exist in the Original field of the current AT1 window.
Cancel: Exit this window without apply the modifications made.
New: Allow to create a new line without close this window.
Validate: Saves the changes made in the autotranslation database persistently, to be valid when required.

In the Skyrim mod "Requiem", are found the following patterns repeatedly:

This tool is useful to externally configure those entries that exist by default in the Main Database that add some new repeated elements to the entries. Look at the examples below for a more real world understanding.

_WINDOW_

After do it, consider to uncheck the Active box of the lines not needed to be applied anymore.

_EXAMPLE_
In the Skyrim mod "aMidianBorn Book of Silence" there is a plugin called "aMidianBorn Content Addon" with multiple lines matching a similar pattern problem.

Gold Ebony Armor
Gold Ebony Armor Eminent Restoration
Gold Ebony Armor Eminent Healty
Gold Ebony Armor Extreme Alteration

Gold Ebony Armor of Invigoration

Auto Translations 5 (AT5)

This is the most versatile and powerful of the autotranslation tools, allowing the user to create and solve many different types of translation patterns quickly and easily. Here is possible to use any number of variables, together with parts of non-variable text. These variables must always be tagged with a number of two digits, even if only one variable is used (e.g Legendary %VAR01%). So, if the pattern needs three variables, they will look like: %VAR01%, %VAR02%, %VAR03%. The order of the variables can be altered in the Translated field (in fact it is common, because syntax differences between languages). So, the previous example can looks like: %VAR01%, %VAR03%, %VAR02%.

The variables used by AT5 work recursively. So, using this variable pattern "%VAR01%, %%VAR02", the tool will try to consider the variables as two parts of; a sentence, and if it does not found the translation in the databases of any of the two variables, will try to translate it again recursively, dividing that part to repeat the search of the variable pattern by groups of words, or just single words. This solves many complex translation situations accurately without the intervention of the user. (Recursivity is explained in detail in the Example 3 below).

The only limitation of the AT5 is that the variables must be separated by something else than a space. ANY character is valid for this indication: a comma ",", an hyphen "-", a carriage return "Enter", etc.

So, the next two examples of these two variables...
%VAR01%, %VAR02%
and...
%VAR01%
%VAR02%
are both correct, because the first is separated by Comma ",", and the second by a Carriage return. However, this example is incorrect...
%VAR01% %VAR02%
because only a Space divides the two variables, and the tool can't know where to delimit them to perform the search in the databases.

A useful tool to use together with AT5, is the utility Ignored Texts (IT), in the next tab of the Auto Translation Options window. The synergy of AT5 and IT, allows the user to resolve automatically special situations in which it is needed to translate some text containing some variables, but with some non translatable text or code that should be ignored, to allow the program automatically translate the variables. This is very helpful to autotranslate potion recipes and similar stuff.

_EXAMPLE 3_ (Recursivity)
NOTE: The {} in this example has no meanings. That's just how the text are in this mod.

In the mod "Valdacils Item Sorting" (the ESP ending as JunkBetter+DEF_INV.esp), also for FO4, can be found this line pattern repeatedly:

"{Scrap} Antique Globe{{{Cork, Plastic, Screw}}}"

To translate it automatically is needed to do it recursively, using AT5. These are the steps:

1. Load the Fallout 4 Main Database in Options/Database/Main Database (if it is not).
2. Add these two lines in the AT5:
Original: {Scrap}%VAR01%{{{%VAR02%}}}
Translated: %VAR01%,%VAR02%
3. Load the mod to translate, select a few lines and press on Apply Autotranslation 5 or or globally by Apply Automatic Translations (F9).
4. The line will be translated automatically (always assuming you have all terms in your database).

The process used by the program to achieve it is this:

1. It detects the first part of the line "Antique Globe", which is in the database, by %VAR01%.
2. It detects the second part of the line "Cork, Plastic, Screw", which is NOT in the Database, by %VAR02%.
3. So with the recursivity, the same %VAR01%,%VAR02% code, will try to translate the non translated part "Cork, Plastic, Screw", assuming the word "Cork" by %VAR01%, and the rest "Plastic, Screw" by %VAR02%. This way, the word "Cork", found in the database, is translated, and the others, not found in the database as a group, not.
4. But again, recursively, AT5 will try to translate these last words, "Plastic" by %VAR01% and "Screw" by %VAR02%. As both are individually in the database, both are translated automatically.

With the previous code, we can translate all the variable text, but not the constant part of the line "Scrap". So, to translate the full line we have two possibilities to choose:

1. Using this code:
Original: {Scrap} %VAR01%{{{%VAR02%}}}
Translated: {Scrap IN YOUR LANGUAGE}%VAR01%{{{%VAR02%}}}
2. By translating the word "scrap" in any database, and using this code:
Original: {Scrap} %VAR01%{{{%VAR02%}}}
Translated: {%VAR03%}%VAR01%{{{%VAR02%}}}

In both cases is needed to press on Apply Autotranslation 5 or or globally by Apply Automatic Translations (F9) to get the lines translated, of course.

Open IME
An IME is a software tool that allows to enter characters and symbols not found in input devices such as keyboards. It is commonly used to insert Chinese, Japanese, Korean and Indic characters for Latin keyboard users. By default, the Microsoft IME versions of these languages are not installed in Windows, and requires doing it and configuring it by the user in Region & Language Settings of the operative system. This is the way:
+ For Windows 7 go to Keyboards and Language, Change Keyboards..., Add..., there select the language to install, press OK, choose it in Default Input Language, OK to close all windows, Log Out and Log In again.
+ For Windows 10 go to Add Language, there select the language to install, Left click on it and press on Set as default, Left click on it again and press Options to revise if Microsoft IME is installed (if not install it from Add Keyboard), Sign Out and Sign In again.
Now you can write in the new language by using your keyboard and the dropdown suggestions offered by Microsoft IME. In some application it starts automatically, but in others like Notepad and ESP/ESM Translator, you need to Right Click on the text frame and press on Open IME to activate it, and Close IME to deactivate it.

Reconversion
Allows the user to type the phonetic of a word using a Western keyboard, to get a dropdown candidate list of characters that represents that word in the active Oriental Language. Usually, it occurs automatically, but this option allows the user to trigger it manually when required.
