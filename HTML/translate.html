主窗口

面板&条形栏


加载了mod的主窗口


主窗口提供了一个舒适的界面，可以完成翻译mod所需的所有基本任务。 执行这些操作的工具列表如下：

  • 菜单栏(Menu Bar)： 提供对程序中包含的所有工具的访问权限。
  • 图标工具栏(Icon Toolbar)： 提供对最常用工具的快速访问。
  • GRUP面版(GRUP Panel)： 提供对加载的mod使用的所有记录组的访问权限，以单独筛选它们。
  • 过滤栏(Filter Bar)： 允许按列快速轻松地搜索和关注所需的文本和单元格。 通过在筛选文本之前添加等号=，将只显示与其完全匹配的行。 例如，如果插入过滤器中的文本是“=good guy”，则所有与“good guy”完全匹配的行都将显示，但其他行（如“you are a good guy”）将被隐藏，因为它除了“good guy”之外还有其他字符/单词。
  • 标签栏(Tabs Bar)： 列出所有打开的MOD以单独翻译，每个MOD都在一个选项卡中，允许在两者之间快速切换。 如果你正在翻译几个mod，这是非常有用的，不需要强迫用户关闭一个mod来处理另一个。 提示： 使用Ctrl+AvPag/RePag在打开的选项卡之间快速切换。(我也不知道AvPag/RePag是什么键位) 单击X红色按钮后会关闭选项卡。每个选项卡还提供了几个实用工具，可在当前翻译中本地使用：
    菜单(Menu)：
    打开mod的另一个视图： 复制实际活动mod的选项卡，允许打开同一mod的多个选项卡。 当你必须同时处理mod的不同部分，或者在远处的词条之间复制/粘贴文本时，这很有用。
    在另一个窗口中打开mod： 为实际活动中的mod选项卡打开一个没有菜单栏(Menu Bar)和图标工具栏(Icon Toolbar)的新窗口。 这对于在mod上工作很有用，可以看到更宽敞的mod视图，但要维护所有翻译所需的基本面板。
    在一个简单的表格中打开mod： 为当前活动中的mod选项卡打开一个新窗口，隐藏除“表格网格”面板及其内容之外的所有工具栏和面板。 适用于快速视觉修订。
    应用此mod已翻译的词条： 按下时，它将触发对选项卡中所有打开的mod的全局分析，搜索匹配的记录，并应用受影响词条的翻译。 此功能取决于下面的两个复选框选项，用于筛选哪些选项卡的MOD接收并/或传播翻译。 实际上，这只是执行该功能的一种可选方式，因为当词条被翻译时，它会自动发生，但有时需要手动执行，比如当你翻译了一个mod的词条，然后加载了另一个具有类似词条的mod时。
    此文件传播其翻译： 如果选中，当前活动mod选项卡中的所有翻译词条将用于翻译打开的其他mod选项卡上的类似词条，当该词条被翻译或按下应用该mod的已翻译词条时。
    此文件可以接收其他打开文件的翻译： 如果选中，当此词条被翻译或按下应用该mod的已翻译词条时， 当前活动的mod选项卡将接收与打开的其他mod选项卡中的已翻译词条所匹配的词条的翻译。
    关闭选项卡： 关闭当前活动的选项卡。
    当前游戏： 允许手动选择mod所属的游戏，以防未自动选择。只需激活复选框并在下拉列表中选择正确的游戏即可。
    当前编码： 允许手动选择mod的文本编码，以防未自动选择。只需激活复选框并在下拉列表中选择对应的文本编码即可。
  • 表格网格面板(Table Grid Panel)： 提供对mod的所有必需数据组件的访问，按列和行排列，以执行翻译。
  • 验证按钮(Validation Buttons)： 允许在启动执行Mod翻译时指定解释每一行的词条。 有四种类型的按钮，可以在显示菜单(Display Menu)中显示/隐藏：
    V - 验证词条： 如果选中，将验证该词条。 这也可以通过编辑菜单(Edition Menu)中的验证所选词条(Validate selected lines)（F10）来完成。
    X - 取消验证词条: 如果选中，则先前验证的词条将不被验证。 这也可以通过编辑菜单(Edition Menu)中的取消验证选定词条(Unvalidate selected lines)（SHIFT+F10）来完成。
    C - 自定义状态(Custom Status Line)： 如果选中，该词条将标记为自定义状态。 该功能只是一个警告标记，用于指示用户决定的任何内容，例如，向翻译团队中的同伴指示哪些词条需要验证。 这也可以通过在编辑菜单(Edition Menu)中将词条状态更改为“自定义”(Put lines status as "Custom")（CTRL+F10）来完成。
    I - 忽略该行(Ignored Line)： 如果选中，该词条将被标记为“已忽略”，以便在触发该模块的翻译时，将其完全隐藏在程序中。 所有忽略的词条将从“主窗口“左侧的GRUP面板中指示的总行数中扣除。 这也可以通过编辑菜单(Edition Menu)中的将词条的状态更改为“忽略”(Define line as ignored)（CTRL+SHIFT+F10）>来完成。 注意： 程序员也使用此功能为上古卷轴4：晨风、上古卷轴4：湮没、辐射3和辐射：新维加斯制作脚本，以忽略其中没有翻译内容的词条。
  • 原文面板： 显示所选单元格的原始文本的大致视图。
  • 译文面板： 显示所选单元格的翻译文本的大致视图。
  • 游戏面板： 显示被翻译mod的游戏图标。
  • 监控面板： 提供选项卡中列出的几个监控工具。
    描述(Comments)： 显示所选行的注释，包含所选文本的完整脚本行以及一些额外的技术数据，如相关任务、NPC派系、链接的媒体文件名等。 用户可以修改它，但要小心脚本。
    关联的mod词条(Linked mod lines)： 提供链接到选定词条所在行的列表。 这在对话框和脚本词条中很常见，它们可以链接到对话的其他行。 这对于满足整体语境的词条翻译是非常有用的。 当选定的词条对应于一本书时，它们也会链接到该书的标题行，反之亦然。
    参照(Suggestions)： 提供一个行列表，其中至少包含一个与要翻译的词条相同的单词作为提示，以了解游戏所用单词/文本的翻译，并将其用作“快速词典”，以翻译具有各种解释的单词/文本。
    词条日志(Line History)： 显示表格网格面板(Table Grid Panel)中所选行中修改的日志历史记录，通过按下右侧的“应用此文本(Apply this text)”按钮，可以恢复以前修改的文本。 之后，文本将在“翻译文本”单元格中更新。
  • Status Bar: Shows two types of statistical data:
    + The number of selected lines, at the start of the lower left corner.
    + The number of lines by color in squares plus the global total after the / in numbers,

    WONDER: By clicking on them, the lines in the Main Window are filtered by color, showing or hidding only the ones chosen or discarded. A Left Mouse Button click on a color square deselects only the color lines picked. A Right Mouse Button deselects all colors except the color picked. To restore all colors to the default selected state, just click with the Left Mouse Button on the global /number_of_lines, at the end of the lower right corner.

TIP: Some of these elements can be shown/hidden via Display Menu.


The Icon Toolbar (and other icons)



1. Open File: Opens a dialog box to select the mod to translate.

2. Launch the Translation of the Mod: Starts the translation of the mod applying the validated lines into the ESP/ESM file. A dialog box will opens to select the location to save the translated mod. A default destination folder for the saves can be set in the Options window for Bethesda games.

3. Options: Opens the Options window to setup the program.

4. Automatic Translations: Opens the Automatic Translations window to work with them.

5. Search & Replace: Opens a window which allows to search and replace some word/text in the Translated Text column. Just insert the search text in Search for and the replacement text in Replace by.

6. Save Translation: Allows to save the current opened mod translation data table as a personal EET/XML database, which can be reloaded to be modified later or used, once complete, as a translation file for different versions or mods.

7. Load Translation: Allows to load a personal EET/XML database previously saved by the Save Translation option above, or downloaded from other users, to be modified or used, once completed, as a translation file for different versions or mods. It´s perfectly viable to load a file which does not correspond with the currently loaded mod, because only the pertinent elements will be used.

8. Export Text: Extracts the raw translated text of the mod to allow the user to copy it globally from any task required. The text to export can be filtered by GRUP.

9. Fast Import: Opens the Fast Import window to work with databases requiring data importation management.

10. Search Main DB: Allows to search, consult, edit the Main Database and apply (Shift+F5) it on the current translation. Also allows to create a new one and merge several databases by importing then.

11. Search Custom DB: Allows to create, search, consult, edit a Custom Database and apply (Ctrl+F5) it on the current translation, as well as merge several databases by importing then.

12. Search OnlyText DB: Allows to create, search, consult, edit an OnlyText Database and apply (Alt+F5) it on the current translation, as well as merge several databases by importing then.

13. Search Current Mod: Offers a quick filter access of the selected text/line in the current mod by F4, to consult and/or edit it without the need to redefine the filters or lose the line focus. Instead, only the Translated Text field and the comment line can be modified. Lines can be validated by F10.

14. Search External DB: Allows to load an External EET/XML/ESP/ESM/TLK/ERF/DAZIP/TP2/SST/PRJ/PEX database to search, consult, or apply (Ctrl+Shift+F5) it on the current translation.

15. Window for Scripts: Allows to open the current selected script line in a window with tools for scripting (full script viewer, syntax highlighting...) to make the translation more informative and secure. Dangerous! Take care when translating parts of scripts because you can produce undesirable changes that can make the game unstable. If you do not know what you are doing, it is better not to translate them. Anyway, the program extracts only the translatable parts of the scripts but not always.

16 . Find Spell Mistake: Allows to find the next spell error in the table grid, scrolling down to it automatically.




1. Apply Automatic Translations: Triggers all the Automatic Translations patterns active to be applied in the current translation.

2. Instant translation of the selected text: Triggers all the Instant Translations of the databases available to be applied in the current translation.

3. See source script: Opens a window with the complete source code of the selected line of the script.

4. Book preview: Opens a book preview window of the selected book line in the Main Window in a readable way, llke in-game.

5. Nifscope - Load model: Opens the mesh (if exsist) of the selected line directly in the program Nifscope. If you entered the right path for game data in Options/Folders, the BSAs from original games will be added as resources in Nifskope, when possible. NOTE: This option is disabled by default and requires to be activated manually in the Miscellaneous tab of the Options window.

6. Play Audio file: Plays the sound files attached to a selected line in mp3/ogg/wmx/wav/fuz formats.




#: Offers a quick access to various tools:
  • Show the line numbers: Shows/hides the line numbers on the left of the text frame.
  • Change font: Allows to change the type, size and other common decorations of the font in the text frame.
  • Dictionary: Allows to select/change the Hunspell spell checker dictionary for the available languages.

The Table Grid Panel

In the Table Grid Panel, there are also some columns showing the different data of each line. Only the Translated Text column is editable, the others are just informative:

  • GRUP: A group of records. Can contains subgroups. It makes the records organized and improves the scanning speed of files.
  • ID: A formID is an unique eight digit hexadecimal code that identifies a record. Various records can share the same name but not the same formID. It is used internally by the game engine and can only be seen and used ingame by the console. Example: 000161E9
  • EDID: A subrecord used as a more descriptive name of a record, to be identified easily by the developers and in some modding tools. Sometimes they are more mnemotecnics, some full name. It is used internally by the game engine, and hidden to the player ingame. Example: SolitudeSawmill.
    NOTE: In Morrowind, the DIAL and CELL GRUPs are the text to translate. This is why it was so hard to translate a mod back in that time
  • Field: A "slot" for a subrecord.
  • Original Text: A non-editable cell containing the text in the original language to translate. It can be modified with the database tools though.
  • Translated Text: An editable cell for the text translated from the original.

It is possible to translate directly in the Translated Text column of the Table Grid Panel (not useful for long text lines though) or in the Translated Text Panel (recommended for long text lines). The cells of the other columns are not editable in this window. To modify them, it is necessary to use the database search tools from the database icons in the Icon Toolbar or from the Databases Menu. These tools are explained later. Also, the program uses various colors to differentiate the way a line has beed translated. To consult the meaning of each color line, go to Filter Menu.

MOD TRANSLATION

IMPORTANT: If the mod contains scripts, the BSA file or the loose .pex files in the Scripts folder must be in the same folder as the .esp/.esm files (better do it outside the Skyrim Data folder), before the mod is opened in the translator program. It should look like this:

File.esp and/or File.esm 		Scripts (folder)
File.esp and/or File.esm

WARNING: Edit the code part of the scripts can produce problems in the game and corrupt the savegame. Only the interface texts of the scripts that are useful for the user, such as messages or MCM menus, should be translated or modified. Usually the program only extracts those parts of the scripts, but not always, so be careful.

ADVICE: Copy the mods to translate in another folder than the Data directory of Skyrim or Fallout. This avoids some problems.


1. Click on Open File (Ctlr+O) and choose the desired ESP/ESM... file to translate.
The program will translate automatically all the text lines that match with the Main Database for the language selected in Options/Databases.
OPTIONAL: There are more possibilities to translate automatically after loading a mod:
A. Using other databases (custom, onlytext, external...) for the text to translate, if exists and are selected in Options/Databases. This is explained below and in the Database Menu section of this tutorial.
B. Loading a previously saved EET/XML translation of the mod, if exists. The program will ask for confirmation or rejection to apply it automatically, if the EET/XML file was saved from that mod previously in the default folder for the saves. This can also be manually by the Load Translation icon (Ctrl+L) in the Translation Menu
C. Loading a different translated version of the ESP/ESM... mod loaded. This is done by pressing Load an already translated mod in the Translation Menu.
D. Using the Auto Translation tools included in the program. They are explained below and in the Auto Translation Menu section of this tutorial.
Tip: All these tools can be used together to complement each other, translating different parts of the mod automatically.
2. Revise visually the automatically translated text cells, and correct manually the errors. There are several tools to help you in this tasks, like the Line Color Filters and SpellCheck Tools.
3. Translate manually the remaining empty cells, if required. Do it in the Translated Text cells of the Main Window.
4. Save Translation (Ctrl+S): Do it regularly during the translation to avoid loosing the work if problems occur.
WARNING: The program uses a temporal autosave, but it is lost if you close the mod loaded without save it manually.
5. Validate Selected Lines (F10): Do it individually by pressing the V button to the right of each line in the Table Grid of the Main Window, or by selecting several or all lines and then pressing on Validate Selected Lines from the Edition Menu or the Contextual Menu, or just pressing the F10 hotkey. Only validated (green) lines are applied in the ESP/ESM... files after pressing Launch the translation of the mod.
OPTIONAL: Unvalidate Selected Lines (Shift+F10): Useful if you forgot to save the translation as EET/XML before validate and in similar cases.
6. Finally, press on Launch the translation of the mod to generate the mod ESP/ESM... file translated, including all validated lines.
DATABASES

NOTE: This section is just an introduction to the program databases. A detailed use of each type of them is found in Database menus. The information found in both sections are complementary, and is recommended to start reading here.


When a mod is loaded by Open File (Ctrl+O) the program starts analyzing it to detect and extract the data for translation, and then compares this data with the various types of databases available, to try to translate instantly the matching text found. Depending on the type of database, the configurations in the Options window, and the way this translation from databases is triggered, some fields in the databases (GRUP, ID, EDID, Original, Translated...) must be required to match with the same fields in the analyzed mod, to allow the program to translate each line in this way. If these requirements are not met, the line/s will not be translated.


The program offers five tools to manage different type of databases. It is possible to have all loaded, if they exist, and then trigger one or various databases depending on the needs of each moment. They are:

Main Databases: This is the database containing all the original and translated texts extracted from a game. It must be downloaded separately from the author's mod page, and copied manually to the BDD folder of the program. It is possible to create your own main databases from the game files (e.g. Skyrim.esm) with the importation tools explained below.
Custom Databases: This database is similar to the Main Database but is offered as an option to have an alternative database created by the user to work in translation. It can be a Main Database modified or a different one, a mixture of two or more databases, or a database created by the user from scratch.
TextOnly Databases: This database is lighter than others and has less features, but allows to contain more data and manage it faster. It is useful to make a temporal quick and short database with some entries to solve some situations in the translation, and also to create or/and load several large databases together.
Mod Databases: This is a second access to the current mod loaded, to quickly filter the selected text/line by F4, for consultation and/or edition without the need to redefine the filters or lose the line focus in the Main Window. Only the Translated Text field and the Comment Line can be modified here though. The lines can be validated by pressing F10.
External Databases: This is a temporal database (cannot be saved) that allows to load ANY type of database (even Primary or Custom), and create a new from many types of game files. It can be used as alternative or complementary database. It also allows to translate mods from old versions that has suffer changes, and prevents the program to identify automatically for this cause.

To create, edit, search or consult them, just click on any of the five colored database icons in the Icon Toolbar, in the Databases Menu or by the hotkeys indicated there. To apply them in the current mod translation, go to the Edition Menu and click on any of the Instant Translation options or by the hotkeys indicated there.


Besides this, when a translation is saved by the Save Translation (Ctrl+S) button, a Personal EET/XML Database is created in the folderBases Persos inside the installation folder of this program. It can be used to continue the translation later, to translate new versions of the mod, and also to share it with other users interested of make its own translation or continue translating an abandoned translation mod. This tasks can be done using a previously translated mod too, using the option Load an already translated mod in the Translation menu.


The program also offers two extra tools to create databases with some features not present in the other five database tools. They are made just to make databases, not to apply them directly in a translation. To do this, it is required to load them in any of the database tools, and then triggered by the Instant Trasnlation in the Edition Menu. Besides, with this import tools is possible to invert the language order of a database (e.g. EN/FR to FR/EN), using the tools Importation or Fast Import in the Databases menu. So, these are the tools:


Fast Import: This tool is similar to the above Importation tool. The finality of both is the same, create databases, but this fast version is more focused to manual edition, and simplifies the analysis and import tasks to make them more intuitive. Also, it adds some new features but lacks others. So, depending on the needs, it is better to use one or the other.
Importation: (Operative but obsoleted by Fast Import)
This is the tool used by the author of this program to create the Main Databases for the games. Also allows to create custom databases from some mods, that can be handy to translate easily after.
AUTOMATIC TRANSLATIONS

Automatic Translations are a set tools implemented by the program to solve quickly and easily some special translations patterns that can be found in some mods. Without these tools the manual work and time to invest would be long, and thus the results are achieved instantly. Some of them are online, some others offline.


ONLINE TOOLS

The program offers several online tools to help find the meaning of individual words or even complete lines of text while translating. When using it for lines or texts, the result is not always accurate, and must be revised visually to avoid mistakes. These are the tools:


Translation by Google Translation (SHIFT+F12):

Translates automatically the selected word/s, text or line/s by the online Google Translate API. It requires registration and configuration in /Options/Miscellaneous/Google Translate API.


Translation by Yandex API (CTRL+F12):

Translates automatically the selected word/s, text or line/s by the online Yandex API. It is configured in /Options/Miscellaneous/Yandex API.


Search in Web the Selection (F12):

This is not an automatic feature, but it is online and similar to Translation by Google Translate. In this case, the selected text or words are opened in your default web browser, and it is translated in the web page of Google Translate. This is useful when you only need to consult the meaning of word/s or phrase/s, but you do not want that the program inserts the text in the active Translated field of the program.


OFFLINE TOOLS

IMPORTANT: These tools should not be confused with other automatic tasks performed by the program like, for example, the analysis comparison and subsequent automatic translation of the matching data found in the databases when a mod isloaded.


There are many types of Automatic Translations tools, but some have become "obsolete" by AT4 and AT5, although they are still usable.


Auto Translations 1 (SHIFT+F9) (AT5 have similar utils): Useful to automatically translate multiple lines matching with a pattern that contains two or three of the elements "TEXT, %VAR%, %PARAM%", in any order. Only one variable and parameter can be used in the same line though. The data can be filtered by GRUP and FIELD.
Auto Translations 2 (CTRL+F9) (Disabled by default, replaced by AT4): Useful to modify the default entries of the Main Database that introduce some new repeated elements. The data can be filtered by GRUP and FIELD.
Auto Translations 3 (ALT+F9) (Disabled by default, replaced by AT5): Useful to indicate the program, new text labels added before or after other text in multiple lines, that exist in the databases, allowing to translate them automatically. The data can be filtered by GRUP and FIELD.
Auto Translations 4 (CTRL+SHIFT+F9): Useful when there are many lines in wich a text element is variable and not found in the databases individually (such as race names), within a sentence in which the rest of the text is constant and found in the databases.
Auto Translations 5 (ALT+F11): Useful to create and solve automatically many different types of patterns quickly and easily, using one or more variables, which must be always tagged with a number of two digits (e.g Legendary %VAR01%). The variables must be separated with any character else than a space to work recursively, and also can be used together with Ignored Texts, to solve some special patterns.
Auto Translation GRUP (CTRL+F11): This automatic translation works like Auto Translation 4, but using the existing value in a GRUP in the databases when is triggered.
Manual Auto Translation (SHIFT+F11): This automatic translation works exactly like Auto Translation 5, but it is easier to use on-the-fly when a pattern occurs just once without the need to be stored.

The program also manages two types of hidden autotranslations, triggered silently when the user translates one line of a group of similar lines with the same text, translating automatically all of them. These features can't be disabled or tweaked in any place of the program though. They are:


Clone Autotranslation: Translates automatically all the lines (in the table grid of the Main Window), which have exactly the same text as the manually translated one. For example, if there are two or more lines with the text "Wasteland Survival", and must be translated manually because is not found in the databases, just translating one of them will translate all the others automatically.

Numerical & Punctuation Autotranslations: Translates automatically the lines matching exactly the same text without accounting any numerical or punctuation variations, after one of the lines is translated manually. It uses the same technique than the without numbers and punctuation dictionaries, but it this case it only uses the text typing text, not the databases. The punctuation feature can be tweaked in the Special Searches tab of the Options window. These are several examples for each of both features:

In both cases, just translating manually one of the lines, like "Wasteland Survival 02" or "It cost from 10% to 20%", will translate all the others automatically, without change the numbers.


Punctuation Pattern
Wasteland Survival!
Wasteland Survival?
Wasteland Survival.
Wasteland: Survival

In this case, all the text in the lines will be translated after translate one of them, without affecting the punctuation symbols, which will remain in place.

HOW_IT_WORKS

The program performs a number of tasks in the background when a mod is loaded, and/or when it works with databases. Depending on the action, some tasks or other are involved in the process in a specific order. These tasks are as follows:


1. Analysis (Mod): When a mod is loaded, the program analyzes its structure, including the scripts if exist, extracting the data required to translate it later, filling the fields of the table grid in the Main Window. At this stage, the program also may ask some questions to the user. For example, if it detects that the mod corresponds to a different game than the selected in the Options, offering the possibility to change it from the pop-up.

2. Comparison (Database): The data extracted from the mod and the patterns included in the auto translation tools, are compared with the active databases (Main, Custom, OnlyText...) and dictionaries (without numbers, punctuation...) when they are triggered.

3. Translation (Table): After the databases comparison is finished, the matching lines in the table grid are automatically translated. This can be tweaked in the Options and AT Options windows, to allow automatic application or the way it is applied.

4. Validation (Line): By default, the program is configured to validate automatically some lines matching some requisites. This can also be tweaked in the Options window, to allow automatic application or the way it is applied.

5. Generation (Mod): This is the last step of a translation, but it is not allowed to be carried out automatically, and needs the user supervision.

TIPS

In this section there are some general tips useful for the translation tasks.


Ways to translate a mod

From a previously saved EET/XML DB: Triggered while opening a mod file (after MessageBox confirmation) or by Load Translation (Ctrl+L).
From a Mod DB (Mod): Triggered from Load an already translated mod in the Translation menu.
From the Main/Custom/TextOnly/External DBs: Edited from any of the Search DB options in the Database menu and triggered globally (F5) or individually from the Instant Translations in the Edition menu.
From one of the five Auto Translation DBs: Edited in the Auto Translation options window and triggered globally (F9) or individually from the Automatic Translations in the Edition menu.
Manually: Translating line to line by the user.


Using Keyboard Shortcuts

The program provides many hotkeys to speed up the most common and repetitive translating tasks. They are indicated in the Menus to the right of each tool name.


Deleting Cells Data

Select the cell/s to erase, go to Edition menu and press Clear Cell (Shift+Del).


Autotranslation Typing Errors Fix

If a typing error occurs while translating a line that triggers any of the autotranslation tools of the program, the other lines affected will keep the error even if the line that caused the problem is fixed. To correct this situation easily, there are some options:
  • If all the checkboxes are checked in the Miscellaneous tab of the Options window, a Modifications List window will open showing all the lines modified by the last action, and allowing to restore them to the previous state by clicking on Back to the previous translation of the selected lines.
  • Use the filter bar in the main window to isolate the affected lines by filtering them with a common word/s. Then select all them in the Translated Text column, and press Shift+Del keys to empty them. Now translate them again without typing error, and all lines will be translated automatically.

Creating a Main DB for Other Languages

This is explained in detail, with various step-by-step examples, in the Fast Import section of this tutorial.


Boosting Production

The knowledge and use of some of the tools in the program, allows the user to solve quickly many of the special translating patterns that can be found during translation. Specially useful are:
  • The Keyboard Shortcuts to accelerate access to the different tools.
  • The Filter bars to search and filter specific lines in many of the tools provided by the program.
  • The GRUP panel to filter, focus and act only on specific group data.
  • The Linked/Suggestions/Comments panels in the Main Window, configurable in the Display menu, to visualize quickly interesting data for the translation.
  • The Search DB tools to search if some specific text/word is in the DBs, know the translation vanilla or edit and expand the databases.
  • The Instant Translation tools to apply the databases in the current mod.
  • The Automatic Translation tools, specially AT5, to translate automatically almost any repetitive special pattern found during translation.
  • The Search and Replace tool, to fix multiline modifications globally.
  • The Comparison tools to find differences between different mod versions, and similarities between different mods.
  • The Original and Translated Characters Filter to filter the visible lines by length.
  • The Miscellaneous submenu tools in the Context Menu to solve some minor tasks easily.

Translation by Group

This is a rare tip for people that wants to translate the dialog and description texts to its native idiom, but prefers to keep, for example, the item and location names in the original English language. To do it, just filter the cell by groups, using the GRUP Panel in the left of the Main Window and translate only the selected group/s.


Dictionaries (Spell Checking & Synonyms)

The program includes support for Hunspell & MS Word dictionaries. It is recommended to use Hunspell because it is free and faster. They are installed in the Dictionaries folder of the installation directory of the program. You can install dictionaries in other languages Hunspell dictionaries for spell checking (.aff & .dic) and synonyms (.dat).
  • Spell Check requires to be checked in the # menu of the Original and/or Translated fields in the Main Window to activate it. After this, the erroneous words will be underlined in red. It is very useful to use the ABC button in the Icon Toolbar or the Find Spell Mistake in the SpellCheck menu, to find the next error.
  • Synonyms help to find words with similar meaning while you are translating. It can be accesses by right clicking on a word in any editable Original, Translated, Custom textbox field of the Main Window and Databases.

All these options can be configured from Configure Dictionaries in the SpellCheck Menu.
