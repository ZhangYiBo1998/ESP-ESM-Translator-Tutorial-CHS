This window contains all configurable options of the program arranged by tabs. The default configurations are right for standard translations. Instead, some of these options can/must also be tweaked when the program is executed for the first time (see Install), or when a mod is loaded (see Mod Translation).

GENERAL OPTIONS

Game: Selects the active game of the mods to translate.
Application Language: Selects the language of the interface of the program.

Encoding
Default input codepage (file to open): Allows to select the character encoding of the file to translate.
Default output codepage (file translated): Allows to select the character encoding of the file translated.
Force input encoding in Fallout 4 and Skyrim SE, which use UTF-8 by default (not advised): This is a exceptional feature made mainly to force the UTF-8 encoding input when using Skyrim LE version of mods in Skyrim SE. It is only available for English mods, because in other languages there are visual character problems. So, this feature is not to "translate" if not to "adapt" mods.
Force output encoding in Fallout 4 and Skyrim SE, which use UTF-8 by default (not advised): This is a exceptional feature made mainly to force the UTF-8 encoding output when using Skyrim LE version of mods in Skyrim SE. It is only available for English mods, because in other languages there are visual character problems. So, this feature is not to "translate" if not to "adapt" mods.
Ask for input encoding each time a file is opened: Allows to enable/disable the popup MessageBox asking the user for the input encoding when a mod is loaded.
Ask for output encoding at each translation: Allows to enable/disable the popup MessageBox asking the user for the output encoding, when the Launch the translation of the mod button is pressed.

Internet Translation
Allows to indicate an URL of the Google Translate web for any language to translate a single selected cell. It can be triggered by the Edition Menu or pressing F12.

Miscellaneous
  • Analyze and translate empty fields: If not checked, the program will hide the lines with the Original Text field empty in the table grid of the Main Window. If checked, the program will show this empty lines, translating them as empty text too.
  • All identical texts are translated: Translates automatically the lines that matches the databases automatically.
  • Automatically validate the line with CTRL+Enter in the edition field of the main window: If active, allows to validate lines by hotkey.
  • Do no add the file CRC at the end of the saved file names. Dangerous! Risk to make a non wanted replacement of the saved file : If checked, the name of the saved EET/XML file (after hit on Save Translation (Ctrl+S), won't include the CRC number of the mod at the end. Not recommended.
  • Replace without confirmation when copy/paste cells: Disables the MessageBox pop up for accept/refuse automatically the overwriting of the cell.
Log of lines
Delete historic after saving: The program stores internally and temporally an history of the modifications made in each line. Checking this box, the program erase all the content stored when a mod translation is saved. This historic of a line can be shown by selecting the line and pressing on Line log in the Display Menu or by right clicking on the line and then on "See modifications done on this line".

Autosave
Number of saves at custom bases: Selects the number of autobackups for user databases.
Automatic saving all XX minute/s (0 = no automatic saving) Allows to indicate the time between automatic saves of the user databases.

Compression level
Zlib compressor level: In a mod, some entries are compressed by zlib. To read one, zlib uses automatically the right numbers, but to save, it is needed to precise. Generally, change the compression level has no consequence, except for the size of the final translated file. But maybe some games can't read a too much compressed field. It never happened to the author though, but the option is there.

Def_grup.xml

Every element in any Bethesda game is considered as a Record, from a sword to an actor or a climate. A GRUP field represents a group of records, and can contain subgroups. It makes the records organized and improves the scanning speed of files. The other Fields are just some slots that contains subrecords of each group.
This window allows to configure which GRUP and Fields will be analyzed by the program when a mod or database is loaded for each game. To select a game, just use the dropdown list at the top left-hand corner. Then, anything can be changed with no danger. In case of big mistake, deleting the files manually will restore the default configuration after restart the program.

The GRUP column contains the list of groups available for the selected game. They can be activated/deactivated by checking the left checkboxes. The Field columns contain the list of fields available for each group, filling them from left to right in a cumulative way. They can also be activated/deactivated by checking the left checkboxes. If Display Max Lengths is checked, a new column will be added at left of each field cell, allowing to indicate the maximum number of characters length of a field. For example, in Morrowind, the CELL are limited to 256 characters. If this value is present, the textbox to type the translation will became red if it is above the 256 characters.

Some GRUP do not contains field analysis (normally only in Skyrim). These are groups which may contain references to scripts. For the program to analyze the scripts, they must have the list of scripts in question. This is an internal list that adds the references to the scripts found after searching for them when analysing the mod. Then, when comes the moment to analyze the scripts, it will do only with the scripts it found. Since few versions, the option exists to analyze all scripts in the scripts folder of the mod. But if a mod analyze in the main data game, the analysis can take time, and the tool will warn you. Concerning the scripts into BSA files, the tools will simply analyze all of them. So just let theses GRUP with no field. They are the GRUPs which can contains scripts reference. The non-listed groups are ignored.

To add a new GRUP or Field subrecord, just write it into the corresponding cell. After completing the changes in the table, it is necessary to press the Save button to apply them. To discard them just press the Cancel button.

FOLDERS

This tab allows to indicate the folder path to save a translated mod after hit on Launch the translation of the mod for each supported game of Bethesda, just by clicking on Browse and choosing a folder in your directory tree for the specific game. To activate/deactivate them, simply check/uncheck the checkbox Create the translated mod file in the game data folder, under the Data Patch of the game.

ADVICE: It is not recommendable to use the original Data folder of the games to avoid some problems.

SPECIAL SEARCHES

Options used when loading a translation save.
List of punctuation characters: Allows to specify the list of punctuation characters used by the internal dictionary without punctuation of the databases.
WARNING: Do NOT add numbers to those fields. Doing that, the numerical auto translation won't works correctly.
The original text must have a maximal length of *** characters:
If the original text length is below this value, the corresponding line won't be added to the dictionary without punctuation.
Remove the finale punctuation of the translated text too: If checked, the final punctuation signs will be excluded if the line to translate is present in a database, but differs only in the final punctuation. The tool will translate only the text and the final punctuation will remain, so both, the text and the final signs will be translated correctly by the program. For example, if a database contains the text hello boy! and the mod to translate contains hello boy. (with a dot, not a "!"), it will be translated by Bonjour mon garçon. with the dot at the end. Without it, it would have been translated with the original punctuation (with the "!").
Automatic Translation Enabled: Enables/disables the Punctuation Autotranslation.

Allows the user to tweak some options to indicate the program the way to apply the dictionary without numbers tool.

MISCELLANEOUS

Show modified lines list when...
IMPORTANT: This tool is dependent on the Automatics options in this same window. There you can activate/deactivate the automatics features individually, and here which of them are showed in the Modifications List panel or window.
IMPORTANT: There are two ways to visualize modified lines. If Display modified line in the Main Window is checked in the Display Menu, the modifications will be shown in the Modifications List Panel in the lower part of the Main Window. If it is not checked, they will be shown in a emergent pop-up window.

This window allows to specify which of the six translation tools checkboxed below, will open a Modifications List window, showing the text changes made in all the affected lines after they are translated at once in the Main Window. It is necessary that more than one line be autotranslated simultaneously to trigger this function. Only the checked translation tool options in this Miscellaneous window will be affected.

The utility is mainly informative, but inside the Modifications List window, the edition of the Translated and Previous text cells are allowed individually, being applied in the Main Window, and the option to Back to the previous translation of the selected lines is offered by pressing the button.

So that, these are the six checkbox tools:

Manual modification: The text introduced manually in the Translated Text cell in the Main Window. Only when there are more than one lines with exactly the same text (numbers are not affected) to the line translated manually, the Modification List window will be open.
Instant translation: The translations triggered from the Instant Translations in the Edition menu. They apply the texts from any of the five selected types of databases.
Automatic translation: The translations triggered from the Automatic Translations in the Edition menu. They apply the texts from the special translation patterns made in the Auto Translation Options window, at the end of the Translation menu or the icon bar.
Translation sentence by sentence: The translation triggered from Translation Sentence by Sentence in the Edition menu. It translates the line selected using the five selected types of databases (main, custom, textonly, current mod and external). If there are other lines with exactly the same text (numbers are not affected), will also be translated. Only in the latter case, the Modification List window will be open.
Apply translation to all corresponding originals: The translation triggered from Apply translation to all same Original Texts in the Edition menu.
Loading a saved mod: The translation triggered from Load Translation and Load an already translated modin the Translation menu.

If you entered the right path for game data in Options/Folders, the BSAs from original games will be added as resources in Nifskope, when possible.
 Executable: This option is for the user to indicate the patch of the program (Nifskope) manually by pressing the Button "...". Obviously, it is required, together with the activation of the previous "Use Nifskope" option, to make the "Nifskope - Load Model" util available to use in the Main Window.

Yandex API
Allows direct online Yandex API translation by selecting word/s, text or line/s and then pressing CTRL+F12 or Translation by Yandex API in the bottom of the Edition menu. The text will be translated automatically, but it is limited to 10.000.000 characters translated by month, and the accuracy is not perfect, requiring revision and correction by the user, but it is generally useful, speeding up the translation process. To activate it, you need to get a free key to use the API on the Yandex website.
 Personal Key: Here you have to introduce a Yandex key. The Get a free API key (on Yandex website) is just a web link to the key.
Input Language: Choose the original language of the text to translate.
Output Language: Choose the language to translate the text.
Confirm these options: Press on this button to validate the changes introduced above.

Google Translate API

The text will be translated automatically, and the accuracy is not perfect, requiring revision and correction by the user, but it is generally useful, speeding up the translation process.
 Input Language: Choose the original language of the text to translate.

Automatics
IMPORTANT: This tool is directly related with the Show modified lines list when... options in this same window. Here you can activate/deactivate the automatics features individually, and there which of them are showed in the Modifications List panel or window.

Here you can enable/disable individually several of the automatic features that offers the program. All them work in the case that several repeated lines exists and are not translated. If one or more of them, but not all, are modified, after the edition is finished, the program automatically will apply the changes to all the other similar lines. The functionality of all this automatics is based on the Clone Autotranslation system.

Enable automatic things (same text, auto translation...) when:
Adding prefix or sufix: If checked, any prefix or sufix added from Add a Prefix or Add a Sufix in Contextual Menu will be automatically added to any other lines with the same text.
Translated Text = Original Text (F8): If checked, when a line is translated by Translated = Original the changes will also be applied automatically to all other lines with the same text.
Instant translation: If checked, when a line is translated by any of the Instant Translation database options, the changes will also be applied automatically to all other lines with the same text.
Text from Main Database/Custom Database: If checked, when a line is translated by Main Database or Custom Database in Contextual Menu, the changes will also be applied automatically to all other lines with the same text.
Automatic translation: If checked, when a line is translated by Automatic Translations, the changes will also be applied automatically to all other lines with the same text.

VERIFICATIONS

Here it is possible to indicate the program a list of forbidden characters by game. If one of these characters is introduced in any text cell of the Main Window while translating, the full Translated field below the Table Grid will be completely colored in red background, and a red text similar to Forbidden Character: [X] will appear below indicating the concrete mistake.

To validate the forbidden characters list introduced for each game, the validation V button at right must be checked. The X button will erase all the content introduced in the correspondent game field. The checkbox Enable the verification of the presence of forbidden characters must also be checked to activate the verification tool globally.
